⚙️ Django Template Engine

The Django Template Language (DTL) — how variables, blocks, loops, and conditions work

📖 Django's Default Template Engine

Django's built-in template engine is a powerful yet simple text-processing system that generates HTML (or any text-based format) dynamically. It follows the MTV (Model-Template-View) pattern, where the template is responsible solely for presentation logic.

🔧 Sandboxed execution — limited Python-like syntax for safety
⚡ Compilation & caching — templates are compiled to Python bytecode and cached
🎨 Inheritance system — DRY principle via blocks and extends
🛡️ Auto-escaping — prevents XSS attacks by default

When render(request, 'template.html', context) is called:

  1. Loader — finds the template file in TEMPLATES['DIRS'] or app directories
  2. Compilation — template text is parsed into Node objects (abstract syntax tree)
  3. Context — the context dictionary (or Context object) is passed to the compiled template
  4. Rendering — each node renders itself using the context, producing the final string
  5. Response — the rendered string is wrapped in an HttpResponse

📦 1. Displaying Context Variables

Variables from the view's context are displayed using double curly braces: {{ variable }}. Django looks up the variable using dictionary, attribute, or list-index access.

View context example:

# views.py
def profile(request):
    context = {
        'username': 'john_doe',
        'age': 25,
        'user': user_object, # object with attributes
        'items': ['apple', 'banana', 'orange']
    }
    return render(request, 'profile.html', context)

Template variable syntax:

<!-- Simple variable -->
<h1>Hello, {{ username }}!</h1>

<!-- Attribute access (dot notation) -->
<p>Email: {{ user.email }}</p>
<p>Full name: {{ user.get_full_name }}</p>

<!-- Dictionary key access -->
<p>{{ user.profile.bio }}</p>

<!-- List index access -->
<p>First item: {{ items.0 }}</p>
<p>Last item: {{ items.2 }}</p>

<!-- Filters: modify variables -->
<p>Uppercase: {{ username|upper }}</p>
<p>Default value: {{ bio|default:"No bio yet" }}</p>
<p>Date formatting: {{ created_at|date:"F j, Y" }}</p>

💡 Note: If a variable doesn't exist, Django renders an empty string (no error).

🧩 2. Template Blocks (Template Inheritance)

Blocks allow template inheritance — you define a base template with {% block %} placeholders, and child templates override them. This follows the DRY (Don't Repeat Yourself) principle.

📄 base.html (parent)

<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}My Site{% endblock %}</title>
</head>
<body>
    <header>Navigation Bar</header>
    {% block content %}
        <p>Default content</p>
    {% endblock %}
    <footer>Footer</footer>
</body>
</html>

📄 child.html (child)

{% extends "base.html" %}

{% block title %}About Us - My Site{% endblock %}

{% block content %}
    <h1>About Page</h1>
    <p>This content overrides the base block.</p>
    {% block extra %}{% endblock %}
{% endblock %}

🔹 Block behavior: Blocks can be overridden, and you can also use {% block.super %} to include parent content.

🔹 Multiple inheritance: You can nest blocks arbitrarily deep. Each block tag takes exactly one argument: the block name.

🔹 Use case: Perfect for consistent layouts (headers, footers, navigation) across many pages.

🔄 3. Loops: {% for ... in %}

The for tag loops over iterable objects (lists, querysets, dictionaries). It provides useful loop variables.

Basic for loop

<!-- Loop over a list of items -->
<ul>
{% for item in items %}
    <li>{{ item }}</li>
{% empty %}
    <li>No items available.</li>
{% endfor %}
</ul>

Loop variables (available inside for loop)

<!-- Access loop metadata -->
{% for user in user_list %}
    <p>
      {{ forloop.counter }}. {{ user.name }}
      <small>({% if forloop.first %}First item{% endif %}{% if forloop.last %}Last item{% endif %})</small>
    </p>
{% endfor %}
forloop.counter — 1-indexed iteration count
forloop.counter0 — 0-indexed iteration count
forloop.first — True if first iteration
forloop.last — True if last iteration
forloop.revcounter — remaining iterations (1-indexed)
forloop.parentloop — parent loop context for nested loops

Looping over dictionaries

{% for key, value in person.items %}
    <dt>{{ key }}</dt>
    <dd>{{ value }}</dd>
{% endfor %}

🔀 4. Conditionals: {% if %} Control Structures

The if tag evaluates conditions and renders content accordingly. Supports elif and else branches.

Basic if/elif/else

{% if user.is_authenticated %}
    <p>Welcome back, {{ user.username }}!</p>
{% elif guest_mode %}
    <p>You're browsing as a guest. <a href="/login/">Log in?</a></p>
{% else %}
    <p>Please <a href="/login/">log in</a> to continue.</p>
{% endif %}

Complex conditions (and / or / not / in)

<!-- Using 'and' and 'or' -->
{% if user.is_staff and user.has_perm 'can_edit' %}
    <a href="/admin/">Admin Panel</a>
{% endif %}

<!-- Using 'in' operator -->
{% if 'admin' in user.groups %}
    <p>Administrator privileges active</p>
{% endif %}

<!-- Checking emptiness -->
{% if not items %}
    <p class="empty">Your cart is empty.</p>
{% endif %}

📌 Available operators: ==, !=, <, >, <=, >=, and, or, not, in, is (for None checks).

⚠️ Note: You cannot use parentheses to group conditions — keep expressions simple.

🔗 Complete Example: All Features Together

<!-- article_detail.html -->
{% extends "base.html" %}

{% block title %}{{ article.title }} | My Blog{% endblock %}

{% block content %}
    <article>
        <h1>{{ article.title|upper }}</h1>
        <p class="meta">Published: {{ article.published_at|date:"M d, Y" }} by {{ article.author.username }}</p>
        
        <div class="content">{{ article.body|linebreaks }}</div>
        
        <h3>Comments ({{ comments|length }})</h3>
        {% if comments %}
            {% for comment in comments %}
                <div class="comment">
                    <strong>{{ comment.author }}</strong>:
                    {{ comment.text }}
                    <small>{{ comment.created_at|timesince }} ago</small>
                </div>
            {% empty %}
                <p>No comments yet. Be the first!</p>
            {% endfor %}
        {% else %}
            <p>Comments are disabled for this article.</p>
        {% endif %}
    </article>
{% endblock %}

📋 Summary: Django Template Engine Flow

Context Variable Display

{{ var.attr }} — dot lookup: dictionary → attribute → list index

Blocks

{% block name %}...{% endblock %} — template inheritance placeholders

For Loops

{% for x in iterable %}...{% empty %}...{% endfor %} — with loop variables

If Conditions

{% if condition %}...{% elif %}...{% else %}...{% endif %} — supports and/or/not/in

⚡ Performance tip: Templates are compiled once and cached in production. Use cached.Loader for better performance.

🛡️ Security: Auto-escaping is enabled by default — use {% autoescape off %} only when necessary.
