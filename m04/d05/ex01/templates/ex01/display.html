🔄 Django request → response cycle
from HTTP request to rendered template (static page)

How a simple template becomes an HTML page: URL resolution, view logic, template rendering, and HTTP response

1. Client Request → 2. URL dispatcher → 3. View (logic) → 4. Template loading → 5. HTML rendering → 6. HTTP Response

🌐 1. HTTP Request

User enters URL /about/ in browser. Browser sends GET request to Django server.

  • Method: GET
  • Headers: User-Agent, Host, etc.
  • WSGI / ASGI server receives raw request
→

🗺️ 2. URLconf (urls.py)

Django looks at ROOT_URLCONF and iterates through urlpatterns to find a matching path.

# urls.py
path('about/', views.about_page, name='about')

Matches /about/ → calls views.about_page

→

⚙️ 3. View function

View receives HttpRequest object. Contains business logic: for static page, prepares context and renders template.

def about_page(request):
    context = {"title": "About Us", "year": 2026}
    return render(request, 'about.html', context)

📁 4. Template loader

render() triggers Django's template engine. It looks for about.html in configured TEMPLATES directories.

  • Checks app_directories loader
  • Filesystem loader if defined
  • Template is loaded as a Template object
→

🎨 5. Template rendering

Template engine processes the HTML + Django Template Language (DTL). Variables {{ title }} are replaced with context values. Tags like {% block %} are evaluated.

<h1>{{ title }}</h1>
<p>Welcome to our static page, year {{ year }}.</p>

→ Generates pure HTML string.

→

📦 6. HttpResponse object

render() returns an HttpResponse instance containing the rendered HTML. Status code = 200 OK. Headers: Content-Type: text/html; charset=utf-8.

  • Optional cookies, cache control
  • Django middleware can process response

📡 7. Final delivery to browser

WSGI/ASGI server sends HTTP response back to client. Browser receives HTML, parses and displays the static web page.

User sees fully styled page — the "static" page is now alive from the simple template.

🗂️ How Django finds the .html file

1. render(request, 'about.html', context)
2. Django loops over template loaders in TEMPLATES['OPTIONS']['loaders']
3. app_directories.Loader searches inside each installed app's templates/ folder
4. Template is compiled and rendered with context

📋 Step-by-step narrative: from request to static page

  1. Client initiates request – Browser sends an HTTP GET to a URL mapped to a template view.
  2. Django's URL resolver – Matches the path against patterns in urls.py and calls the associated view.
  3. View execution – The view receives request, defines context, and calls render().
  4. Template loading & rendering – The engine loads the .html file, parses template tags ({% ... %}), replaces variables ({{ var }}), and generates HTML string.
  5. HttpResponse construction – render() wraps the HTML into an HttpResponse object.
  6. Middleware post-processing – Response middleware adds headers or performs compression.
  7. Web server returns response – Server sends HTTP response with status 200 and HTML body. Browser renders the static webpage.

✨ Key insight: Even a "static" page from a template goes through the full MTV (Model-Template-View) cycle – but without database models. The view acts as the orchestrator, and the template layer produces the final markup.

⚡ Simplified sequence (pseudo-code flow)

1. Browser GET /about/ 2. Django URL dispatcher → matches 'about/' → views.about 3. about(request): context = {'page': 'static_content'} return render(request, 'static_page.html', context) 4. Template engine: loads static_page.html → replaces {{ page }} → produces HTML string 5. Wrap in HttpResponse(content=html_string, status=200) 6. Send to client
⚙️ Django's request-response lifecycle — concise explanation. Every static/dynamic page follows URL → View → Template → Response pipeline.
💡 Remember: The "static" in static web page refers to the final HTML not being dynamically generated by client-side JS, but it is still dynamically assembled on the server from templates.
